# List Export & Sharing Pattern
## As implemented in the Miniatures Collection & Army Builder

This document describes the export and sharing pattern used in `minis.tashefamily.com` for use as a reference when implementing the same feature in other apps.

---

## Overview

Two export mechanisms are provided on every saved list:

1. **PDF Export** — generates a clean, print-friendly PDF client-side using jsPDF, with no server involvement
2. **Shareable Link** — encodes the list as a compressed URL parameter, allowing read-only viewing without authentication

Both are implemented in `js/export.js` and called from the lists view. They share the same data input — a normalized list object — making the module reusable across all games.

---

## PDF Export

### Library
**jsPDF** (`https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js`) — client-side only, no server needed, MIT licensed.

### Implementation approach

```javascript
// js/export.js
function exportListToPDF(list) {
  const { jsPDF } = window.jspdf;
  const doc = new jsPDF();

  // Header
  doc.setFontSize(18);
  doc.text(list.name, 14, 20);

  doc.setFontSize(11);
  doc.setTextColor(100);
  doc.text(`${list.gameId} · ${list.faction} · ${list.totalPoints} ${list.pointsLabel}`, 14, 28);

  // Divider
  doc.setDrawColor(200);
  doc.line(14, 32, 196, 32);

  // Units
  let y = 40;
  list.units.forEach(unit => {
    doc.setFontSize(12);
    doc.setTextColor(0);
    doc.text(`${unit.unitName}${unit.pilotName ? ` — ${unit.pilotName}` : ''}`, 14, y);

    doc.setFontSize(10);
    doc.setTextColor(80);
    doc.text(`${unit.points} pts`, 180, y, { align: 'right' });
    y += 6;

    // Upgrades
    if (unit.upgrades?.length) {
      unit.upgrades.forEach(upg => {
        doc.setFontSize(9);
        doc.setTextColor(120);
        doc.text(`  ${upg.upgradeName} (${upg.points})`, 18, y);
        y += 5;
      });
    }
    y += 3;

    // Page overflow guard
    if (y > 270) {
      doc.addPage();
      y = 20;
    }
  });

  // Strategy note
  if (list.strategy) {
    y += 4;
    doc.setDrawColor(200);
    doc.line(14, y, 196, y);
    y += 8;
    doc.setFontSize(10);
    doc.setTextColor(60);
    const lines = doc.splitTextToSize(list.strategy, 176);
    doc.text(lines, 14, y);
  }

  // Footer
  doc.setFontSize(8);
  doc.setTextColor(160);
  doc.text('Generated by minis.tashefamily.com', 14, 290);

  doc.save(`${list.name.replace(/\s+/g, '-').toLowerCase()}.pdf`);
}
```

### Key design decisions
- **Client-side only** — no server round-trip, works offline
- **Page overflow guard** — long lists with many upgrades can exceed one page; the `y > 270` check adds a new page and resets position
- **Filename** — derived from list name, spaces replaced with hyphens, lowercased
- **Font sizes** — 18pt title, 11pt subtitle, 12pt unit names, 10pt upgrade names, 9pt details

## Shareable Link

### Approach
The list is serialized to JSON, compressed with LZ-string, and base64-encoded into a URL parameter. The recipient loads the URL, decodes the parameter, and renders the list read-only without needing to authenticate.

### Library
**LZ-String** (`https://cdnjs.cloudflare.com/ajax/libs/lz-string/1.4.4/lz-string.min.js`) — lightweight compression for strings, MIT licensed. Reduces URL length significantly for large lists.

### Implementation

```javascript
// js/export.js

// Generate shareable link
function generateShareableLink(list) {
  // Strip any fields not needed for read-only display
  const exportPayload = {
    name: list.name,
    gameId: list.gameId,
    faction: list.faction,
    totalPoints: list.totalPoints,
    pointsLabel: list.pointsLabel,
    units: list.units,
    strategy: list.strategy,
    exportedAt: new Date().toISOString()
  };

  const json = JSON.stringify(exportPayload);
  const compressed = LZString.compressToEncodedURIComponent(json);
  const url = `${window.location.origin}/shared?list=${compressed}`;

  // Copy to clipboard
  navigator.clipboard.writeText(url).then(() => {
    showToast('Link copied to clipboard');
  });

  return url;
}

// Decode shareable link (called on /shared page load)
function decodeSharedList(encodedParam) {
  try {
    const json = LZString.decompressFromEncodedURIComponent(encodedParam);
    return JSON.parse(json);
  } catch (e) {
    return null; // Invalid or corrupted link
  }
}
```

### Read-only shared view (`/shared`)
A lightweight page that:
1. Reads the `?list=` URL parameter on load
2. Decodes and parses the list using `decodeSharedList()`
3. Renders the list in a clean read-only layout (no owned/wishlist toggles, no save button)
4. Shows a "View in app" link prompting the viewer to install/open the app
5. Handles decode failure gracefully with a "Link invalid or expired" message

### URL length consideration
For a typical X-Wing squad (~5 units, some upgrades), the compressed+encoded payload is around 400–600 characters — well within URL limits. A large Armada fleet with many upgrades might reach 1,200–1,500 characters, still safely under browser and server URL length limits (~2,000 characters is the practical safe limit for broad compatibility).

If a list is unusually large and the URL would exceed ~1,800 characters, show a warning: "This list is too large for a shareable link — use PDF export instead."

## The normalized list object

Both export methods consume the same input shape. This is the normalized list object passed to both functions:

```javascript
{
  name: string,               // "Rebel Alpha Strike"
  gameId: string,             // "xwing-2e"
  faction: string,            // "Rebel Alliance"
  totalPoints: number,        // 197
  pointsLabel: string,        // "Points" | "BV" | "PV" | "Level"
  pointLimit: number,         // 200
  gameType: string,           // "points-based" | "level-based"
  units: [
    {
      unitName: string,       // "X-Wing"
      pilotName: string,      // "Luke Skywalker" (null for level-based)
      upgrades: [             // empty array for games without upgrades
        { upgradeName: string, points: number }
      ],
      points: number          // unit total including upgrades
    }
  ],
  strategy: string,           // AI-generated strategy note
  aiGenerated: boolean,
  exportedAt: string          // ISO timestamp (shareable link only)
}
```

## How to adapt for another app

To use this pattern in a different app:

1. **Copy `js/export.js`** — the two functions (`exportListToPDF` and `generateShareableLink`) are self-contained
2. **Load the CDN scripts** in your HTML:
   ```html
   <script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
   <script src="https://cdnjs.cloudflare.com/ajax/libs/lz-string/1.4.4/lz-string.min.js"></script>
   ```
3. **Normalize your data** to the list object shape above before calling either function — the PDF layout and URL encoding both expect this structure
4. **Create a `/shared` route** in your app to handle the read-only view
5. **Adjust the PDF layout** — the column widths, font sizes, and field labels in the PDF function are easy to customize for different content types. For non-list content (e.g. a board game collection export), restructure the unit loop to match your data shape.

### What to change for a non-list app
The shareable link mechanism works for any JSON-serializable data — just replace the list object with whatever payload your app needs to share. The compression and encoding/decoding functions are generic. The PDF function is the part that needs the most adjustment for different content types — the current layout assumes a hierarchical list (name → units → upgrades), which won't suit tabular or freeform content without restructuring.
